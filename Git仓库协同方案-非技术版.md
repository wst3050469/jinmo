## 📖 什么是Git？

**简单解释**：
Git就像一个"云文档库"，可以保存项目文件的所有历史版本。每次修改都会被记录下来，可以随时查看或恢复到之前的版本。

**类比**：
- 就像你的文档有"另存为"功能，但Git更强大
- 就像Google Docs的"版本历史"，但更专业
- 就像游戏的"存档"功能，可以随时读取之前的存档

---

## 🎯 为什么使用Git？

### ✅ 优势
1. **简单易用**: 不需要复杂的技术配置
2. **版本管理**: 可以查看每次修改的历史记录
3. **代码共享**: 开发AI和部署AI可以访问同一个代码库
4. **问题追溯**: 出了问题可以查看是谁、什么时候修改的
5. **安全可靠**: 代码保存在云端，不会丢失

### 📊 工作流程
```
开发AI（在云端工作）
    ↓ 提交代码
Git仓库（云端存储）
    ↓ 通知部署AI
部署AI（下载代码并部署）
```

---

## 🔧 需要什么？

### 1. Git账号
- 需要一个代码托管平台的账号（如GitHub、GitLab、Gitee）
- 免费，注册简单

### 2. 仓库地址
- 创建一个新的Git仓库后，会得到一个地址
- 格式类似：`https://github.com/用户名/scms-project.git`

### 3. 访问权限
- 部署AI需要"读取"权限（可以下载代码）
- 开发AI需要"写入"权限（可以上传代码）

## 🚀 如何操作（简单步骤版）

### 阶段一：创建Git仓库（一次操作）

#### 步骤1: 创建GitHub账号（如果没有）
1. 访问 https://github.com
2. 点击"Sign up"注册
3. 填写用户名、邮箱、密码
4. 验证邮箱

#### 步骤2: 创建新的仓库
1. 登录GitHub后，点击右上角的"+"号
2. 选择"New repository"
3. 填写仓库信息：
   - **Repository name**: scms-project
   - **Description**: 施工综合管理系统
   - **Public/Private**: 选择Private（私有，更安全）
4. 点击"Create repository"
5. 复制仓库地址（类似：`https://github.com/用户名/scms-project.git`）
6. **把这个地址告诉我（部署AI）**

#### 步骤3: 配置访问权限
1. 点击仓库的"Settings"选项卡
2. 左侧菜单选择"Collaborators"
3. 点击"Add people"
4. 输入部署AI的用户名（我告诉你）
5. 给予"Write"权限

---

### 阶段二：开发AI如何使用

#### 📋 日常操作（每次开发完成后）

**1. 保存代码到Git**

开发AI会在完成工作后，执行以下命令（自动）：
```
git add .
git commit -m "完成营销推广模块开发"
git push
```

**解释**:
- `git add .` → 标记所有修改的文件
- `git commit -m "完成营销推广模块开发"` → 创建一个版本记录
- `git push` → 上传到云端仓库

**2. 触发部署**

开发AI提交代码后，会通知部署AI：
```
代码已上传到Git仓库，版本：v1.0.1
请部署AI下载并部署
```

---

### 阶段三：部署AI如何使用

#### 📋 日常操作（收到通知后）

**1. 下载最新代码**

部署AI会执行以下命令：
```
git pull
```

**解释**:
- `git pull` → 从云端下载最新代码

**2. 部署到服务器**

部署AI会将代码部署到生产服务器（120.55.5.220）

**3. 验证部署结果**

检查服务是否正常运行

## 🔍 如何查看代码历史？

### 查看提交记录
1. 打开Git仓库地址
2. 点击"Commits"选项卡
3. 可以看到每次提交的：
   - 提交时间
   - 提交人
   - 修改说明
   - 修改了哪些文件

### 查看某个版本的代码
1. 点击某个提交记录
2. 可以查看该版本的完整代码

### 恢复到之前的版本
如果新版本有问题，可以恢复到之前的版本

## 💡 常见问题解答

### Q1: 代码会丢失吗？
**A**: 不会。Git会保存所有历史版本，永远不会丢失。

### Q2: 可以多人同时工作吗？
**A**: 可以。但最好约定好，避免同时修改同一个文件。

### Q3: 如何知道谁修改了什么？
**A**: Git会记录每次修改的详细信息，包括时间、修改人、修改内容。

### Q4: 如果修改错了怎么办？
**A**: 可以恢复到之前的版本。

### Q5: 需要付费吗？
**A**: 免费版本足够使用。私有仓库免费。
